# White Rose SEO Implementation Roadmap

Phased implementation plan for moving White Rose from a one-page static site to a focused SEO foundation for specialist engineering consulting.

## Phase 1: Foundation, Weeks 1-4

Goal: Make the site technically complete, measurable, and ready for service-page expansion.

| Task | Owner | Output | Success Criteria |
| --- | --- | --- | --- |
| Verify HTTPS redirect | Technical | HTTP redirects to `https://whiterose.no/` | No HTTP version indexable |
| Connect Search Console | Technical | GSC property for `whiterose.no` | Sitemap submitted and crawled |
| Confirm GA4 events | Technical | Email click and outbound click tracking | Organic lead actions visible |
| Add schema | Technical | Organization, ProfessionalService, WebSite, Person | Rich Results test passes where applicable |
| Publish `/services/` | Content | Service hub page | Indexed and linked from homepage |
| Publish `/about/` | Content | Detailed Bjarne profile | Person schema and sameAs links present |

## Phase 2: Expansion, Weeks 5-12

Goal: Build commercial landing pages and the first supporting informational pages.

| Task | Output | Success Criteria |
| --- | --- | --- |
| Publish architecture review page | `/services/architecture-review/` | Page explains process, deliverables, fit, and CTA |
| Publish technical direction page | `/services/technical-direction/` | Page targets CTO/founder decision support |
| Publish DevSecOps page | `/services/devsecops/` | Page covers deployment, automation, security workflows |
| Publish development page | `/services/software-development/` | Page covers implementation and handover |
| Publish infrastructure automation page | `/services/infrastructure-automation/` | Page supports platform and operations intent |
| Publish first two checklists | `/insights/architecture-review-checklist/`, `/insights/codebase-review-checklist/` | Each links to a relevant service page |
| Update sitemap | `sitemap.xml` includes new URLs | URLs submitted and indexed |

## Phase 3: Scale, Weeks 13-24

Goal: Build authority and proof around the service pages.

| Task | Output | Success Criteria |
| --- | --- | --- |
| Add proof hub | `/work/` | Clear entry point for project examples |
| Add 2-3 proof pages | Anonymized or named project examples | Each includes problem, approach, result, and constraints |
| Publish monthly insight posts | 4-6 posts | Each supports a commercial page |
| Start link building | Outreach and profile updates | 5-10 relevant mentions or referring domains |
| Add FAQ sections | Service page FAQs | FAQ content answers buyer objections naturally |
| Review CWV | Lighthouse and field data | All templates pass Core Web Vitals targets |

## Phase 4: Authority, Months 7-12

Goal: Turn White Rose into a trusted specialist entity for senior engineering review and implementation support.

| Task | Output | Success Criteria |
| --- | --- | --- |
| Publish original technical essays | 6-8 high-quality posts | Search impressions and referrals grow |
| Publish deeper case studies | 2-3 detailed proof pages | Leads cite proof pages in conversations |
| Add comparison or fit pages | Pages explaining when White Rose is the right or wrong fit | Better qualified inquiries |
| Build external authority | Guest posts, directories, podcasts, GitHub, partner mentions | Referring domains and branded search grow |
| Refresh top pages | Updated service pages based on GSC data | CTR and rankings improve |
| Monitor AI visibility | Manual checks or tool-based tracking | White Rose appears for relevant expert queries |

## Dependencies

| Dependency | Needed For |
| --- | --- |
| Search Console access | Keyword, indexing, and CTR decisions |
| GA4 access | Lead and engagement measurement |
| Hosting access | HTTPS redirect, headers, deployment verification |
| Project examples | Case studies and proof pages |
| Public profile links | Person schema and E-E-A-T |
| Content decisions | Whether to target Norway-only, English-only, or bilingual pages |

## Risk Mitigation

| Risk | Mitigation |
| --- | --- |
| Publishing too many weak pages | Use quality gates before adding pages to sitemap |
| Competing with large consultancies on broad terms | Target specialist and high-intent terms first |
| Thin service pages | Include deliverables, process, fit, examples, FAQ, and contact path |
| No external authority | Prioritize technical articles and trusted profile links |
| Analytics without action | Review metrics monthly and decide one concrete change per review |

## Monthly Review Checklist

- Check indexed pages in Search Console.
- Review top queries by impressions and clicks.
- Check organic landing pages in GA4.
- Review email clicks and qualified inquiries.
- Update sitemap when new pages go live.
- Refresh one underperforming page if impressions are rising but CTR is weak.
- Record new backlinks, citations, or public mentions.
